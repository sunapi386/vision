# Build pipeline

## Overview

The book is authored in MDX files organized by chapter, then compiled into static HTML pages. There are two output pages:

- **`book.html`** -- the full book with audiobook player, table of contents, and paragraph-level audio sync
- **`index.html`** -- the condensed essay version at jasonsun.org

Both are generated by `build.py`.

## Content structure

Each chapter lives in its own directory (`part1-world-changed/`, `part2-the-void/`, etc.) containing:

- `index.mdx` -- chapter introduction (the first `# Heading` is the chapter title)
- `01-section.mdx`, `02-section.mdx`, ... -- numbered sections in reading order

The build script walks these in sorted order, converting markdown to HTML with the `tables`, `fenced_code`, `footnotes`, and `smarty` extensions.

## Paragraph tagging

Every `<p>` element in the book gets an `id="ab-{chapter}-{index}"` attribute, where `chapter` is 1-6 and `index` is 0-based within the chapter. This is done by `tag_paragraphs()` in build.py. These IDs are the anchor points for audiobook synchronization -- clicking a paragraph seeks the audio to that paragraph's timestamp.

Headings (`<h1>`-`<h6>`), list items, tables, and other elements do not get `ab-` IDs.

## Audio alignment

`build.py` calls `align_paragraphs_to_timestamps()` which loads pre-computed timestamps from `timestamps/{slug}.json`. Each JSON file is an array where index `i` corresponds to paragraph `ab-{ch}-{i}`:

```json
[
  [0.0, 12.34],
  [12.94, 28.56],
  null,
  [29.16, 45.02]
]
```

A `null` entry means that paragraph had no audio (synthesis failed). These timestamps are generated by `tts.py`.

The alignment data is embedded as a JS object (`apAlignments`) in the book HTML. The audio player uses it to highlight the current paragraph and enable click-to-seek.

## Templates

Shared assets live in `templates/`:

- `shared.css` -- all styles shared between book and essay pages
- `shared.js` -- theme toggle, scroll animations, analytics beacon
- `nav.html` -- navigation bar with active-page highlighting

The book page has its own extensive template (the `TEMPLATE` string in build.py) that includes the audio player, table of contents sidebar, and paragraph sync JS.

## Running the build

```bash
python3 build.py
```

This produces `book.html` and `index.html`. The audio files and timestamps must already exist (generated by `tts.py`) for the audiobook to work.

## Chinese version

The Chinese translation lives in `zh/` and is built by a separate script (`build-vision-zh.py` in the parent directory). It follows the same structure but uses the `zm_yunjian` voice for TTS.
